# Kodeai

**The AI-first full-stack TypeScript framework.**

Kodeai is a framework where AI is the primary developer. Instead of adapting human frameworks for AI, Kodeai is architecturally designed so AI knows exactly what to generate, where to put it, and how to validate it.

```
Idea → Spec (YAML) → kodeai generate → Working API with database
```

## Quick Start

```bash
kodeai init my-app
cd my-app
kodeai dev
```

That's it. You now have a REST API with full CRUD, SQLite database, pagination, and timestamps — from a single YAML spec.

```bash
# Try it
curl http://localhost:3000/api/tasks
curl -X POST http://localhost:3000/api/tasks \
  -H "Content-Type: application/json" \
  -d '{"title":"Buy groceries","priority":"high"}'
```

## How It Works

**1. Write a spec:**

```yaml
# specs/models/product.spec.yaml
model: Product
fields:
  id:
    type: uuid
    primary: true
  name:
    type: string
    max: 200
  price:
    type: decimal
    min: 0
  stock:
    type: integer
    default: 0
  active:
    type: boolean
    default: true

api:
  endpoints: [list, get, create, update, delete]
  auth:
    list: public
    create: authenticated
    delete: admin
  sortable: [name, price, createdAt]
  pagination:
    defaultLimit: 20
    maxLimit: 100
```

**2. Generate:**

```bash
kodeai generate
```

From ~25 lines of YAML, Kodeai generates:

- TypeScript types and interfaces (`Product`, `CreateProductInput`, `UpdateProductInput`)
- Drizzle ORM database schema with proper column types, constraints, and indexes
- Zod validators with all constraints (min, max, enum values, etc.)
- Hono REST API with CRUD, pagination, error handling
- Auto-generated header with source tracking on every file

**3. Run:**

```bash
kodeai dev
```

Server starts with SQLite (zero config), auto-creates tables from specs, and serves your API.

```
Kodeai Dev Server

  ✓ Database: ./dev.db
  ✓ Table: products
  ✓ Server running at http://localhost:3000

  Routes:
    GET    /api/products
    GET    /api/products/:id
    POST   /api/products

    DELETE /api/products/:id
```

## What Gets Generated

From a single spec, `kodeai generate` produces:

| File | Content |
|---|---|
| `models/product.types.ts` | `Product`, `CreateProductInput`, `UpdateProductInput`, `ProductResponse`, `ProductListResponse` |
| `models/product.schema.ts` | Drizzle `pgTable` with uuid, varchar, decimal, boolean, timestamps |
| `validators/product.validator.ts` | `CreateProductSchema`, `UpdateProductSchema`, `ProductListQuerySchema` with all Zod constraints |
| `api/product.controller.ts` | Full CRUD: list (paginated), get, create, update, delete with error handling |
| `api/product.routes.ts` | Hono routes wired to controller |
| `app.ts` | Hono app with CORS, error handler, health check, route registration |
| `db.ts` | Database connection |

Every generated file includes an auto-generated header:

```typescript
/**
 * AUTO-GENERATED by Kodeai — Do not edit manually
 * Source: specs/models/product.spec.yaml
 * Generated: 2026-03-26T19:34:08.865Z
 */
```

## Spec Features

### Field Types

`string`, `text`, `integer`, `decimal`, `boolean`, `date`, `timestamp`, `uuid`, `json`, `enum`, `email`, `url`, `slug`, `password`, `file`

### Constraints

```yaml
name:
  type: string
  min: 1              # Minimum length
  max: 200            # Maximum length
  unique: true        # Unique constraint
  index: true         # Database index

price:
  type: decimal
  precision: 10
  scale: 2
  min: 0

status:
  type: enum
  values: [draft, published, archived]
  default: draft

password:
  type: password
  min: 8
  hidden: true        # Never included in API responses
```

### Relations

```yaml
relations:
  author:
    type: belongsTo
    model: User
    foreignKey: authorId
    onDelete: cascade

  comments:
    type: hasMany
    model: Comment

  tags:
    type: manyToMany
    model: Tag
```

### API Configuration

```yaml

    list: public
    create: authenticated
    update: owner
    delete: admin

```

## CLI Commands

```bash
kodeai init [name]        # Create new project with example spec
kodeai generate [target]  # Generate code from specs (models, api, or all)
kodeai validate [spec]    # Validate specs without generating
kodeai dev                # Start dev server with SQLite + auto-migrate
```

## Why Kodeai?

- **66% of developers** spend more time fixing AI-generated code than writing it from scratch
- AI code has **1.7x more bugs** than human-written code
- Current frameworks were designed for humans — AI has to guess conventions

Kodeai fixes this:

| Principle | How |
|---|---|
| **Spec-driven** | Describe what you want in YAML. The framework generates correct code. |
| **Type contracts** | TypeScript types constrain output — invalid code won't compile. |
| **Absolute conventions** | Every file has one correct location. Zero ambiguity. |
| **Immediate validation** | `kodeai validate` catches errors before generation. |
| **AI-native** | `CLAUDE.md` auto-generated so any AI understands the project instantly. |

## Architecture

```
┌─────────────────────────────────────────────┐
│           SPEC LAYER                        │  ← .spec.yaml (you write this)
├─────────────────────────────────────────────┤
│           ENGINE LAYER                      │  ← Parser + Validators + Generators
├─────────────────────────────────────────────┤
│           RUNTIME LAYER                     │  ← Hono + SQLite + Zod
├─────────────────────────────────────────────┤
│           OUTPUT LAYER                      │  ← Generated code (don't edit)
└─────────────────────────────────────────────┘
```

## Tech Stack

| Layer | Technology | Why |
|---|---|---|
| Language | TypeScript (strict) | Types constrain AI output |
| HTTP | [Hono](https://hono.dev) | 14KB, TypeScript-first, edge-compatible |
| Database | SQLite via [libsql](https://github.com/tursodatabase/libsql) | Zero config for dev |
| Validation | [Zod](https://zod.dev) | Auto type inference, declarative |
| Schema | [Drizzle](https://orm.drizzle.team) | Type-safe, real SQL, no magic |
| Testing | [Vitest](https://vitest.dev) | Native ESM + TS, ultra-fast |
